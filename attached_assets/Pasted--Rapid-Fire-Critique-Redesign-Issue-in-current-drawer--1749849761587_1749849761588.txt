### Rapid-Fire Critique → Redesign

| Issue in current drawer                                           | Why it hurts                                        | Fix                                                                                                                             |
| ----------------------------------------------------------------- | --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| **50 % / 100 %** traffic numbers buried in gray under “Variants”. | Hard to spot uneven allocation (looks like a typo). | Move traffic split to a pill next to variant label:<br>`Control • 50 %`  `Variant B • 50 %`<br>Color text when split ≠ planned. |

## Redesigned Drawer (wireframe)

```plain
┌─────────────────────────────────────────────────────────────────────────┐
│  New Booster UX                running 🟢    Δ +2.3 pp   p 0.07   Pow 65⚠ │
│  1 / 7 days  ▬▬▬▬░░░░░  ETA 7 d                                       │
├─────────────────────────────────────────────────────────────────────────┤
│ Variants                 Traffic     Metric (Engagement Score)         │
│ ● Control                50 %        42 %                              │
│ ● Variant B              50 %        44.3 %   (+2.3 pp)                │
│                                                                       │
│  tiny sparkline here  ┈┈╱╲╱╲╱                                          │
│                                                                       │
│  Actions:  [ Pause ]  [ Stop Early ]         ⋮                         │
└─────────────────────────────────────────────────────────────────────────┘
Overflow (⋮): Duplicate • Delete • View History
```

### Key Elements

1. **Pulse Cluster** (Δ, p, Power) together; color rules:

   * Δ text green/red when p < 0.05; gray otherwise.
   * Power < 50 % gray, 50–79 % yellow “⚠”, ≥ 80 % green check.

2. **Progress Bar** right under title (fills by elapsed ÷ planned days). Thin 6 px line, brand blue.

3. **Countdown Pill** shows dynamic ETA; turns orange if ETA > planned test length (indicates under-powered).

4. **Mini Sparkline** (primary metric delta day-by-day). Hover reveals daily values; keeps row height \~160 px.

5. **Actions Row**

   * Primary = Pause (ghost)
   * Danger = Stop Early (red, only enabled if Δ negative & p < 0.05)
   * Overflow menu for edge actions.

6. **Keyboard/Screen-reader**

   * Entire drawer focusable; arrow keys navigate variants table; buttons have aria-labels (“Pause test Booster UX”).
